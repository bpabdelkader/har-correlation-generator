## 📖 What is this project?

This **Spring Boot** web application takes **HAR files** (HTTP Archive) recorded while navigating scenarios in **Mozilla Firefox** and does the heavy lifting:

- Parses every request & response
- Automatically detects **dynamic variables** (session IDs, tokens, CSRF values, timestamps, UUIDs, query parameters, headers, JSON bodies, etc.)
- Creates **ready-to-use correlation rules**
- Displays every variable and its value in a colored pattern

Perfect for anyone who scripts load tests with **JMeter**, **Gatling**, **k6** or any other tool where manual correlation is necessary to replay the recorded scenario

---

## ✨ Key Features
- ✅ **Drag-and-drop HAR file upload**
- ✅ **Intelligent correlation engine (regex, JSONPath, header/body extraction)**
- ✅ **Interactive correlation table showing variable extraction and reuse points**
- ✅ **One-click export of correlation rules (JSON / JMeter `.jmx` snippet / Gatling Scala)**
- ✅ **Side-by-side request viewer with variable highlighting in source traffic**
- ✅ **Advanced filtering and search (by variable name, domain, or request type)**
- ✅ **Export of fully parameterized HAR file with dynamic variables**
- ✅ **Conversion of the scenario into a ready-to-import JMeter `.jmx` file**

---

## 🚀 How to use it (in 30 seconds)

1. Record your scenario in Firefox
   → Open DevTools → Network tab → “Export HAR” (or use the **HAR Export Trigger** extension)

2. Run the Spring Boot app (`mvn spring-boot:run`)

3. Open `http://localhost:8080`

4. Drop your `.har` file → get instant correlations!

5. Copy the rules or export directly into your test script.
